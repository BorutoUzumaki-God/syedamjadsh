IoTSecure is a firmware analysis and vulnerability monitoring dashboard used by security teams to detect threats in IoT devices. The workflow was redesigned to simplify complex scan data into clear compliance reports, severity breakdowns, and actionable insights.

View Prototype

Overview

A cybersecurity platform designed to help organizations monitor, assess, and secure their IoT devices. The prior system worked functionally, but the experience was inefficient, visually inconsistent, and difficult to navigate, which led to slower response times and missed security risks.

The goal was to rethink the information hierarchy and create a clear, insight-first dashboard that helps security teams make faster, more confident decisions.

Problem Summary

IoT devices such as CCTV cameras, access control systems, and sensors are often shipped with weak firmware security, outdated software versions, and default passwords. This makes them an easy entry point for attackers.

High-risk alerts blended into dense tables, compliance visuals lacked meaning, and the navigation scattered related tasks across multiple screens. Teams could detect vulnerabilities — but struggled to respond to them.

The result: time lost in triaging risks, which increases the window for breaches.

Research & Insights

To understand user friction, I conducted:

  • Review of existing UI usage patterns
  • User interviews with security analysts and network admins
  • Competitive analysis (Hart, Claroty, Tenable One)

Key Insights

  • Security teams scan multiple devices daily
  • Alerts lacked visual prioritization
  • Too many screens for core actions
  • Technical data presented without simplification

Problems This Caused

  • Slower interpretation of results
  • Longer time to decide what to fix first
  • Scattered workflows and higher cognitive load
“Make critical risks obvious first. Show context only when needed.”

Stakeholder Workshops & Ideation

Workshops with:

  • CTO and Engineering Leads
  • Head of Product
  • Security Operations Analysts

We aligned on three guiding principles:

  • Clarity over density
  • Risk-first visibility
  • Structured drill-down (Summary → Category → Details → Suggested Fix)

User Flow

The system was redesigned into a logical, layered flow that mirrors how analysts actually think during investigation.

Wireframes to High-fidelity

Low-fidelity layouts emphasized structure—clean hierarchy, clear spacing, and digestible modules. After validating interaction logic, I moved into high-fidelity visual design:

  • Neutral dark UI tone to match cybersecurity theme
  • Strong highlight colors for risk severity
  • Cards and contextual graphs for data
  • Simplified navigation with fewer mental jumps

Prototyping, Testing & Iteration

The interactive prototype was tested with:

  • 3 SOC Engineers
  • 2 Network Security Admins

Impact

  • Faster vulnerability triaging
  • Lower learning curve for new analysts
  • Clear awareness of what needs immediate attention
  • Modernized, trustworthy enterprise look
